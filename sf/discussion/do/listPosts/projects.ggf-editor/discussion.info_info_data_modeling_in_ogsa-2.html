Nitpicking
Though a good paper overall, the process of converting it to PDF seems to have badly affected several of the figures, 
and a few others need tweaking:
  Fig 1: The top bubble is wholly black!
  Fig 2: Should have something in the space in the middle or it looks like a mistake (a cloud? a question mark?) rather 
than a preparation for Fig 3.
  Fig 4: Something is wrong with the transparency of the figures in the VOs.
  Fig 5: Would it be possible to increase the font sizes so that the text is easier to read when printed? And move the "
matching" text so that it is not covering/covered by the line it labels.
  Fig 7: Consider highlighting/making bold the terms that will be picked up by the queries in Figures 8-11.
  Fig 11: Make the font a little smaller so that none of the lines wrap where not desired (yes, I find this a pain in 
the documents I write too...)

Also, the contributors list probably ought to have the addresses of the organizations for which the authors work, and 
the contributor acknowledgments probably shouldn't use shortened first names.

(Finally, don't forget to fix the GFD references, of course.)

Re: Nitpicking
I generated a pdf using a free facility called pdf995....and my responses are based on that....to summarize, the pdf 
problems you describe are not seen with this converter.  I will talk with Joel Replogel and Greg Newby to see what pdf 
converter they use to make sure  doc converts corrects to pdf.

Figure 1 I removed the pattern.

Figure 2 I inserted a grayed out dotted line bubble with a question mark as you suggest.

Figure 4 I left alone because the transparency is right.

Figure 5 I increased the font size where possible without having to redraw the entire diagram.

Figure 7 I think the highlighting you suggest will be too confusing to the reader (even if we state that in text) and 
make the XML less readable.  The reader can alway use query functions to find terms.

Figure 11 is already at 9 pitch and we don't want to make it any smaller.

The authors chose to not include the explicit org addresses (very long)..email addresses are most useful.  In the 
acknowledgments section, I'm not aware that shortened first names have been used so I don't know what to fix (and these 
are names that have been used in other docs as-is).

For Snelling's first name, I changed 'Dave' to David'.

I fixed the GFD references.  Changed the model reference to the assigned GFD# and for the GLUE reference changed it to 
point to the GLUE drafts because Glue is not expected to be published until late Aug/Sep.

 	Hiro Kishimoto (2005-09-27 02:32:22)

	
	

I have one meta comment on this document. This document normatively
 depends on "Usage Record - XML Format." But "Usage
 Record" is still working draft stage. It may be ok to start
 public comment review period but I think it should not be published
 until usage record spec is also published.

	
		
	John Tollefsrud (2005-09-26 17:55:31)

	
	

I've read this document and consider it appropriately developed
 to advance to public comment. I recommend the authors consider
 in a single cycle Olle's observations along with other comments
 as the document may receive during the public comment period. 

jt

	
		
	Olle Mulmo (2005-09-12 11:25:47)

	
	

I have one concern about section 5.5 in that it seems a
 bit overworked. In addition to the extractRUSUsageRecord(XPath)
 operation you also have several
 extractThisParticularAttributeGivenThatParticularKey()
 operations.

This seems redundant and superfluous: it is as if you have written
 the spec with a particular (non-documented) operational setting
 in mind, or made assumptions what type of queries (and frequency
 thereof) that people will perform on the service.

AFAICT, with XPath I can easily construct all of the queries
 you have provided different operations for. Also, in the query
 I can indicate a reduction of the result set so that only the
 attributes that I care about are returned.

In short, I'm not an advocate of adding convenience methods to
 a recommendations document unless there is a very strong motivation
 behind it.
